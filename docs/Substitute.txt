Substitute

Syntax: #substitute {message} {newmessage} {priority}

The #substitute command allows you to change mud output. The %1-99 variables are substituted from the message and can be used
in the newmessage part of the substitute. The %0 variable shouldn't be used in the message part. The priority part is
optional and determines the priority of the substitute, it defaults to 5.

See the action entry for more information about the available regular expression options. You can add tintin color codes in
the newmessage part to change the color of the message, default all the previous color codes of the mud message are stripped.
See Colors for more information.

Example: #substitute {%1 tells you '%2'} {<128>%1 <138>tells you '<128>%2<138>'}
Add this substitution and give yourself a tell to see the result.

Notice: You can remove substitutions with the #unsubstitute command.
